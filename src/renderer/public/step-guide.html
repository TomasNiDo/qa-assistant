QA Assistant Step Writing Guide

Write one step per line. For all DOM-targeting actions, use explicit locator suffixes so the parser and generated Playwright code reflect exact intent.

Explicit Locators Ambiguous Steps = Warning (Not Blocking) Default Code View = Full Playwright Test

Quick Rules

  • Use one action per step line.
  • Quote values and targets: "Search", "headphone".
  • DOM actions should end with using <locator-kind>.
  • Go to, Expect, and dialog actions do not need locator suffixes.
  • Ambiguous legacy steps still run, but show rewrite guidance.

Locator Vocabulary

  • label: using label
  • placeholder: using placeholder
  • role: using role button
  • text: using text
  • testid: using testid
  • css: using css
  • id: using id
  • class: using class

Canonical Syntax

Enter

Fill input

Enter "<value>" in "<field>" field using <locator-kind>
Click

Click target

Click "<target>" using <locator-kind>
Select

Select dropdown option

Select "<option>" from "<dropdown>" dropdown using <locator-kind>
Checkbox

Check / Uncheck

Check "<label>" checkbox using <locator-kind>
Hover

Hover target

Hover over "<target>" using <locator-kind>
Press

Press key (optional target)

Press "<key>" in "<field>" field using <locator-kind>
Upload

Upload file(s)

Upload file/files "<paths>" to "<input>" input using <locator-kind>
Network / Download

Wait after click

Wait for request "<METHOD pattern>" after clicking "<target>" using <locator-kind> within 10s
Wait for download after clicking "<target>" using <locator-kind> within 15s

Non-Element Actions (No Locator Suffix)

Go to /checkout
Expect "Successfully added to cart" within 10s
Accept browser dialog
Dismiss browser dialog

Generated Playwright Code Shape (Default)

import { test, expect } from '@playwright/test';

test('Add headphone to cart', async ({ page }) => {
  // Enter "headphone" in "Search" field using placeholder
  await page.getByPlaceholder('Search').fill('headphone');

  // Click "Search"
  await page.getByRole('button', { name: 'Search' }).click();

  // Click "Wireless Bluetooth Headphone"
  await page.getByRole('button', { name: 'Wireless Bluetooth Headphone' }).click();

  // Click "Add to Cart"
  await page.getByRole('button', { name: 'Add to Cart' }).click();

  // Expect "Successfully added to cart" within 10s
  await expect(page.getByText('Successfully added to cart')).toBeVisible({ timeout: 10000 });
});
Legacy shorthand (for example Click "Search") remains supported and runnable. The app now surfaces an inline warning with a deterministic explicit rewrite suggestion.
