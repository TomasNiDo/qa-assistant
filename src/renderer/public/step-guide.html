QA Assistant Step Writing Guide

Write one step per line. Use quotes around values and targets. For the most reliable runs, use strict grammar formats shown below.

Quick Rules

  • Use one action per step line.
  • Prefer strict syntax instead of free-form natural language.
  • Use full or absolute file paths for uploads when possible.
  • For element selectors, use class or ID forms when text matching is unstable.
  • Use timeouts on network/assertion/download steps if the action is slow.

Core Step Types

Enter Text

Fill an input field

Use this to type values into forms such as login, search, checkout, OTP, and profile fields.

Enter "<value>" in "<field>" field

  • Enter "john@example.com" in "Email" field
  • Enter "123456" in "OTP" field
Click

Click buttons, links, menu items, or text targets

Use delayed clicks when UI transitions or animations need a short pause first.

Click "<text>" or Click "<text>" button after 2 sec

  • Click "Sign in"
  • Click "Continue" button after 1 sec
Navigate

Move to a path or URL

Supports absolute URLs and project-relative paths.

Go to <path-or-url> or Redirect to <path> url

  • Go to /checkout
  • Go to https://example.com/help
Expect

Assert visible text

Add timeout clauses for slower pages to avoid premature failures.

Expect <assertion> with optional within 45s

  • Expect dashboard is visible
  • Expect "Order placed" within 45s

Selector-Based Click Targeting

Use these formats when text-based clicking is unstable or ambiguous.

Class Selector

Target by class name

Click element with '<className>' class

  • Click element with 'btn-finish' class
ID Selector

Target by ID

Click element with this ID '<elementId>'

  • Click element with this ID 'submitButton'
Identifier

Target generic identifier

Click element with '<identifier>'

  • Click element with 'btn-finish'
Lookup behavior: class and ID forms are treated as selectors. Generic identifiers are attempted in this order: data-testid, data-test-id, id, then button/link/menu/text.

Advanced Step Types

Select Dropdown

Select "<option>" from "<dropdown>" dropdown

  • Select "United States" from "Country" dropdown
Checkbox

Check "<label>" checkbox or Uncheck "<label>" checkbox

  • Check "Remember me" checkbox
  • Uncheck "Subscribe to newsletter" checkbox
Hover

Hover over "<target>"

  • Hover over "Profile"
Keyboard

Press "<key>" optionally in "<field>" field

  • Press "Control+K"
  • Press "Enter" in "Search" field
Upload

Upload file/files "<paths>" to "<input>" input

  • Upload file "C:/fixtures/invoice.pdf" to "Receipt" input
  • Upload files "C:/a.png,C:/b.png" to "Attachments" input
Browser Dialog

Accept browser dialog or Dismiss browser dialog

Enter "<text>" in prompt dialog and accept

  • Enter "CONFIRM" in prompt dialog and accept
Wait For Request

Wait for request "<METHOD pattern>" and expect status "200"

Wait for request "<pattern>" after clicking "<target>" within 10s

  • Wait for request "POST **/api/login" and expect status "200"
  • Wait for request "**/api/profile" after clicking "Load profile" within 10s
Wait For Download

Wait for download after clicking "<target>"

  • Wait for download after clicking "Export CSV" within 15s

Natural-Language Fallback and Troubleshooting

The parser accepts many natural phrasings (such as Tap the continue button, Visit /orders, Type 123456 into OTP field), but strict syntax is more deterministic.

  • If parse preview fails, convert the step to one of the strict formats above.
  • If click targets are unstable, switch to class/ID selector forms.
  • If assertions or network waits fail intermittently, add explicit timeout clauses.
  • If upload fails, verify the file path exists on the machine running the app.
